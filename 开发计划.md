# OPPO Watch ColorOS Watch x3 — 168 轻断食 App 开发计划

> 计划生成日期：2026-06-27  
> 目标设备：OPPO Watch x3（ColorOS Watch，圆形表盘）  
> 开发方式：原生 Android APK，通过 ADB 侧载安装  
> AI 辅助模型：DeepSeek V4 Pro + Thinking 模式（你指定的配置）

---

## 一、技术选型

| 项目 | 选择 | 说明 |
|------|------|------|
| 开发语言 | **Kotlin** | 原生 Android 首选，无需依赖 Wear OS |
| UI 框架 | **Jetpack Compose**（标准版） | ColorOS Watch 不支持 Wear OS Compose，用标准版手动适配圆屏 |
| 最低 SDK | **API 26 (Android 8.0)** | 兼容性安全 |
| 目标 SDK | **API 33 (Android 13)** | 当前最优 |
| 构建工具 | **Gradle + Kotlin DSL** | Android Studio 默认 |
| 布局策略 | **ConstraintLayout + Canvas 自定义绘制** | 圆屏适配核心 |

> ⚠️ **关键**：ColorOS Watch **不是 Wear OS**，不能用 `wear-compose`、`Horologist`。所有圆屏适配手动完成。

### 圆屏适配要点

| 要点 | 方案 |
|------|------|
| **圆环** | `Canvas.drawArc()` 匹配物理圆角 |
| **安全裁剪** | `Modifier.clipToBounds()` + 圆形 Outline |
| **文字** | 以圆心为基准，不靠边 |
| **按钮** | 圆角大按钮，大拇指可及区域 |
| **字体** | 最小 16sp |
| **触摸** | 最小 44dp |
| **方向** | 只有纵向，简化 |

### 色彩方案

| 用途 | 颜色 | 色值 |

## 六、启动优化策略

ColorOS Watch x3 性能有限，启动速度是核心体验。

| 优化项 | 措施 | 预期收益 |
|--------|------|----------|
| **去 Splash** | 直接加载主界面 | 省 ~300ms |
| **View 懒加载** | `LaunchedEffect` 异步初始化 Service | UI 即现 |
| **禁用 MultiDex** | 方法数 < 64K | 省 ~200ms |
| **矢量图标** | 全程 Vector Drawable | APK < 1.5MB |
| **ProGuard** | shrinkResources | APK < 1MB |
| **Service 延迟** | 先渲染 UI，再绑定后台 | 首帧 < 100ms |
| **预加载** | `android:windowBackground` 纯黑 | 无白闪 |

**预期冷启动：< 500ms**

## 十、已知坑 & 对策

### ColorOS Watch x3 限制

| 问题 | 对策 |
|------|------|
| 不支持 Google Play | 用 NotificationCompat + 振动，不用 FCM |
| 双芯片架构，后台可能被杀 | 前台 Foreground Service + 持续通知 |
| 屏幕小触控延迟 | 纯 Canvas 绘制，不搞复杂动画 |
| 如果未来锁 ADB | 走 OPPO 开发者通道申请签名 |
| 圆形屏幕安全区 | `onLayout` 计算裁剪边界 |
